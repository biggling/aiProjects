## 3. MCP SDK / Anthropic Updates — April 6–12, 2026

Source: [daily1bite.com/en/blog/ai-tutorial/claude-code-april-2026-update] and [releasebot.io/updates/anthropic/claude-code]

**Claude Code April 2026 Highlights:**

- **`/powerup` command** — Interactive animated tutorial system teaching Claude Code features in-session.
- **MCP Tool Result Size: 500K chars** — The `_meta["anthropic/maxResultSizeChars"]` annotation now allows up to **500,000 characters** per tool result (previously 25K hard limit). This is a major unlock for data-heavy MCP tools like crypto portfolio dumps, DB schemas, or tax basis exports.
- **Permission Deferral ("defer")** — `PreToolUse` hooks can now return a `defer` decision instead of allow/block. Headless sessions pause and resume with `-p --resume`, allowing human-in-the-loop at a specific tool call without aborting the whole session.
- **`CLAUDE_CODE_NO_FLICKER=1`** — Flicker-free rendering for terminal output; useful for long-running servers.
- **Streamable HTTP is now the unambiguous standard** — SSE transport officially unsupported in new tooling. All new MCP servers must use Streamable HTTP only.

**Key implication for BiG:** The 500K char tool result limit removes a major constraint for portfolio/tax MCP servers that need to return large transaction histories. The previous 25K limit would have required pagination workarounds.

## 4. Stripe MPP / x402 / Google AP2 — Real Deployments

Source: [workos.com/blog/x402-vs-stripe-mpp-how-to-choose] and [stripe.com/blog/machine-payments-protocol]

### Who Has Actually Shipped?

**Stripe MPP (live since March 18, 2026) — confirmed live deployments:**
- **Browserbase** — Agents pay per headless browser session via MPP.
- **PostalForm** — Agents pay to send physical mail via MPP.
- Stripe now bundles MPP inside the "Agentic Commerce Suite" alongside ACP and x402 support.

**x402 — indie deployments:**
- x402 joined the Linux Foundation (April 2, 2026), signaling governance maturity but still early for real deployments.
- WorkOS analysis confirms x402 is the better choice for **indie/open API** use cases — no accounts, no signup, HTTP-native payment.
- No confirmed public indie MCP server has publicly disclosed x402 revenue yet, but the infra is live. [defiprime.com/stripe-mpp-vs-x402]

**Google AP2:**
- Still an orchestration layer spec, no confirmed MCP server integrations from indie developers as of this writing.

**Decision guidance for BiG:**
- Use **x402** if building an open-source-adjacent server with usage-based pricing (pay-per-API-call).
- Use **Stripe MPP / Lemon Squeezy** if building a subscription product targeting Claude Pro users (simpler billing, better fraud protection).
- For the crypto tax MCP — **Lemon Squeezy subscription** remains the cleaner path given Thai VAT requirements.

## 5. New High-Score Opportunity Scan

### Legal Research / Case Law MCP
- Multiple implementations exist: **Mahender22/legal-mcp** (4M+ US court opinions, PACER, Clio), **CourtListener official MCP** [courtlistener.com/help/mcp/], **JamesANZ/us-legal-mcp**, **agentic-ops/legal-mcp**.
- **Verdict: Too crowded, and US-centric. Low BiG fit (geographic mismatch).**

### Real Estate MCP
- Several products already live: **Apify/NexGenData Real Estate MCP** (Redfin data), **BatchData MCP** (skip tracing + comps), **Cotality MCP** (property + location intelligence), **sap156/zillow-mcp-server** (Zillow data).
- **Verdict: Crowded with well-funded data providers. Not a gap.** The Thai real estate data niche (Dot Property, DDproperty) has zero MCP coverage but also limited WTP and small TAM.

### Sports Betting MCP
- Already exists: **The Odds API MCP** (two separate implementations on PulseMCP by kitchenchem and Wagyu Sports Team), **SportsGameOdds MCP**, **balldontlie-api/mcp** (NBA/NFL stats).
- **Verdict: Covered for US sports betting. NOT a gap.** A **Thai-specific sports betting API MCP** (covering SEA leagues, Muay Thai, local bookmakers) has zero coverage but faces serious regulatory risk in Thailand.

### Emerging Gap Found — Crypto P&L + Tax (Reinforced)
All three adjacent areas (legal, real estate, sports betting) are occupied. The crypto tax/cost basis niche remains the only unoccupied high-WTP niche with strong API availability (CCXT, Etherscan, CoinGecko, blockchain RPCs) and confirmed buyer pain (1099-DA per-wallet requirement active January 1, 2026).
