### Etherscan / BSCScan / Polygonscan — Rate Limits

- **Free tier:** 3 calls/second, 100,000 calls/day across all chains. Returns up to 10,000 records per call for normal transactions.
- **Important change (late 2025):** Etherscan restricted free API access on several chains — Avalanche, Base, BNB Chain, and Optimism are no longer fully covered on free tier. ~90% chain coverage remains free.
- **Paid tier:** Starts at $49/month (Pro) — unlocks richer on-chain data and higher call limits.
- **Alternative explorer emergence:** Following Etherscan's free tier cuts, block explorer alternatives (Blockscout, etc.) have gained traction among indie developers.

[docs.etherscan.io/resources/rate-limits] [info.etherscan.com/whats-changing-in-the-free-api-tier-coverage-and-why]

**So what?** Etherscan free tier covers Ethereum mainnet (the highest-value chain for tax purposes) without issues. Use Moralis for BSC/Base/Avalanche to avoid Etherscan's restricted chains. $49/month Pro only needed if serving hundreds of users with large wallets.

### TaxBit and Koinly — Public APIs?

- **TaxBit:** Has an internal API for data import; does not publish a public developer API or SDK that third parties can wrap. No documented MCP integration.
- **Koinly:** Supports CEX integration via read-only API keys (user-side OAuth to exchanges, not a Koinly developer program). Bitget recently upgraded its API integration with Koinly. No public API for external developers to query Koinly accounts.
- **Bottom line:** Neither TaxBit nor Koinly offers a wrappable public API. They are end-consumer products, not data platforms.

[coinledger.io/tools/taxbit-vs-koinly] [bitget.com/blog/articles/bitget-revolutionizes-crypto-tax-reporting-by-enhanced-api-integration-with-koinly]

**So what?** No risk of a "Koinly MCP" that directly competes by wrapping their own API. BiG's MCP competes on being the tool layer that LLMs can actually call — TaxBit/Koinly remain GUI products with no agent-accessible API surface.

### CEX Trade History Without User OAuth

- **Binance:** Discontinued historical trade data via API before September 2022 (effective November 2024). The official API limits history to the last 3 months. The only workarounds are: (1) CCXT with user API keys, (2) browser automation with session cookies via `binancecrawler`, (3) user-exported CSV.
- **Coinbase:** Advanced Trade API requires OAuth2 or API keys — no anonymous historical access.
- **Universal pattern:** Every CEX requires user-provided API keys (read-only) or CSV export. There is no "scrape CEX history without user credentials" solution that is both legal and reliable.

[anchorbrowser.io/hub/binance-data-export-automation-api] [docs.ccxt.com]

**So what?** The MCP must ask users to provide their exchange API keys (read-only). This is standard for all crypto tax tools — it is not a friction disadvantage vs competitors. CCXT handles key-based auth for 100+ exchanges in one library.

## 2. MCP Server Architecture Patterns

### Production Boilerplates Worth Forking

Three open-source templates directly address auth + billing + rate limiting:

**1. iannuttall/mcp-boilerplate** [github.com/iannuttall/mcp-boilerplate]
- Cloudflare Workers-based remote MCP server
- User authentication built in
- Stripe integration for paid tools (metered usage + subscription)
- Best fit for BiG's use case: deploy on Cloudflare (free tier), charge via Stripe

**2. chrisleekr/mcp-server-boilerplate** [mcpservers.org/servers/chrisleekr-mcp-server-boilerplate]
- TypeScript + Streamable HTTP (correct 2026 transport)
- OAuth Proxy via Auth0, Dynamic Application Registration
- OAuth session storage built in
- Heavier setup; better for multi-tenant enterprise use cases

**3. f/mcp-startup-boilerplate** [github.com/f/mcp-startup-boilerplate]
- Rails-based (not TypeScript — less relevant for BiG)
- OAuth2 provider + Stripe payments included
- Useful for reference architecture even if not forked directly

### How Successful Servers Handle Key Patterns

| Pattern | Common Approach (2026) |
|---------|----------------------|
| User API key storage | Encrypted at-rest in Cloudflare KV or Postgres; user provides via tool call or setup flow |
| Rate limiting | Gateway layer (Cloudflare, Bifrost, or custom middleware); per-user token bucket |
| Caching | Cloudflare KV for price data (TTL 60s); in-memory LRU for session-scoped data |
| Billing gate | Feature flag checked against subscription record; Stripe webhook updates user tier in DB |
| Multi-user isolation | User ID derived from OAuth token; all DB queries scoped by user ID |

**MCP Gateway options for rate limiting:** Bifrost (Go, open-source, no licensing cost), Kuadrant/mcp-gateway (Kubernetes-native). For a solo indie server, Cloudflare Workers' built-in rate limiting is sufficient.

[lunar.dev/post/the-best-open-source-mcp-gateways-in-2026] [composio.dev/content/best-mcp-gateway-for-developers] [mintmcp.com/blog/mcp-gateways-rate-limiting-access-control]

**So what?** Fork `iannuttall/mcp-boilerplate` as the starting skeleton. It already has Cloudflare + Stripe wired together, which matches BiG's zero-infra-cost constraint. Estimated setup time to working paid server: 1–2 days.

### Thai Crypto Tax — The 5-Year Exemption (Critical Finding)

Thailand enacted a **personal income tax exemption on crypto capital gains from January 1, 2025 through December 31, 2029** — but ONLY for trades on Thai SEC-licensed platforms (Bitkub, Orbix/Satang, etc.).

Key details:
- **0% capital gains tax** on crypto sold via SEC-licensed Thai exchanges through 2029.
- **Still required:** Users must calculate gains, declare them on **PND.90 or PND.91** forms (standard Thai income tax return), and mark as exempt. Records must be retained for 5 years.
- **NOT exempt:** Staking rewards, mining income, airdrops, DeFi yield — these remain taxable as income.
- **Off-platform trades** (Binance, Coinbase, DEXs, P2P) are NOT covered by the exemption — gains are taxable at personal income tax rates (0–35%).

[expattaxthailand.com/thailand-crypto-tax-exemption-2025-2029] [chiangraitimes.com/finance/thailand-crypto-tax-guide-2026] [nishimura.com/en/knowledge/publications/20250701-113956]

**So what?** The exemption is a double-edged sword: Thai users trading only on Bitkub have reduced urgency for a tax tool for capital gains — but staking/DeFi/Binance income is still fully taxable and still requires cost basis calculation. The pain point shifts to: (1) off-platform traders (large Binance user base in Thailand), (2) DeFi/staking income calculation, (3) record-keeping for the exemption itself (still need proof of transactions). The product is still viable; messaging must target these segments.

### Bitkub API — Developer Access

Bitkub has an **official public REST API and WebSocket API**, documented on GitHub and api.bitkub.com:
- **Public (no auth) endpoints:** Market tickers, order books, trade history, depth — all available without API keys.
- **Private (auth required) endpoints:** User balance, order placement, trade history — require API Key + Secret (user-generated from Bitkub dashboard).
- **Current version:** V3 endpoints (V1 deprecated).
- **No formal developer program** or MCP integration found — open field.

[github.com/bitkub/bitkub-official-api-docs] [api.bitkub.com]

**So what?** Bitkub's read-only private API (user API key) enables importing Thai CEX trade history directly. This is a meaningful differentiator — no other crypto tax MCP supports Bitkub. Add Bitkub as a first-class exchange on launch.
